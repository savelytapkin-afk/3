# Документация исправлений и проверок v3.5

## ✨ Улучшения v3.5

### 1. Поле «Макс. писем» (max_letters) в UI
- **Проблема**: лимит писем хранился только в `settings.json` и не отображался в интерфейсе
- **Исправление**: добавлено поле «Макс. писем» рядом с «Задержкой» в разделе ⚙️ УПРАВЛЕНИЕ
- **Поведение**: `0` = обрабатывать все письма; любое положительное число — лимит

### 2. Подстановка `{title}` в первом письме
- **Проблема**: шаблон поддерживал `{price}` и spintax, но не `{title}`
- **Исправление**: добавлена замена `{title}` на название товара из данных парсера

### 3. Подстановка `{name}` в ответе
- **Проблема**: шаблон ответа поддерживал `{link}` и `{price}`, но не `{name}`
- **Исправление**: добавлена замена `{name}` на имя продавца в `check_and_reply`

### 4. Retry-логика в `dolphin_start`
- **Проблема**: при временной недоступности Dolphin Anty соединение не восстанавливалось
- **Исправление**: 3 попытки с паузой 3 сек при `ConnectionError`; явный `RuntimeError` при 401/403

### 5. Логирование ошибок `generate_link_via_api`
- **Проблема**: при ошибке CreateAd API функция молча возвращала пустую строку
- **Исправление**: ошибки и пустые ответы теперь выводятся через `print` (видны в консоли)

# Документация исправлений и проверок v3.4

## 🐛 Исправление: "invalid session token" (v3.4)

### Проблема
При нажатии "▶ СТАРТ" появлялась ошибка `invalid session token` и профили Dolphin Anty не открывались.

### Причины
1. **Устаревший endpoint** — код обращался к `POST /v1/browser/start-browser`, который больше не поддерживается
2. **Токен без нормализации** — пробелы и переносы строк из конфига попадали в HTTP-заголовок `Authorization`
3. **Нет валидации** — токен не проверялся перед массовым открытием профилей
4. **Нет обработки 401/403** — ошибки авторизации не диагностировались явно

### Исправления
- Endpoint обновлён: `POST http://localhost:3001/v1.0/browser_profiles/{id}/start`
- Добавлен `.strip()` при загрузке (`_load_config`), сохранении (`save_token`) и использовании токена
- Добавлен метод `_validate_dolphin_token()` с тестовым GET-запросом к `/v1.0/browser_profiles`
- В `_open_profiles`: при HTTP 401/403 выводится явное сообщение и цикл прерывается
- Добавлена retry-логика: 3 попытки с паузой 3 сек при `ConnectionError`
