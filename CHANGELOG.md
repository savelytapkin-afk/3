# 📝 История изменений

## v2.2 (2026-06-25)

### ✨ Новые возможности
- ✅ **Две отдельные вкладки** - Отправка писем и Проверка ответов
- ✅ **Профили остаются открыты** между этапами (в памяти `self.drivers_pool`)
- ✅ **Уникальный HTML для каждого ответа** - каждому свой адрес от CreateAd
- ✅ **Персонализация по email** - {link}, {price}, {name}
- ✅ **Новый этап Проверка ответов** - нажал кнопку, проверил ответы, отправил HTML
- ✅ **Сохранение данных продавцов** - все email с оригинальными данными хранятся
- ✅ **Новая функция generate_personalized_html()** - генерирует уникальный HTML
- ✅ **Новая функция _check_and_reply_v2()** - работает с открытыми профилями
- ✅ **Новая функция _create_ad_via_api()** - создаёт объявления в CreateAd

### 🗑️ Удалено
- ❌ **Валидация email** - берём готовые из парсера
- ❌ **Функция validate_email()**
- ❌ **Автоматическое закрытие профилей** - пользователь сам закрывает

### 🎨 UI улучшения
- Новая вкладка "💬 Проверка ответов"
- Информация о статусе профилей в вкладке проверки
- Разделение параметров между двумя этапами
- Кнопка "🔍 ПРОВЕРИТЬ ОТВЕТЫ" вместо авто-проверки

### 📋 Архитектура

**Этап 1: Отправка писем**
```
Парсинг email-ов
↓
Открытие профилей Dolphin
↓
Отправка первых писем (ротация профилей)
↓
Профили ОСТАЮТСЯ ОТКРЫТЫ
```

**Этап 2: Проверка ответов** (когда пользователь нажмёт кнопку)
```
Проверка входящих писем
↓
Для каждого ответа:
  - Находим email отправителя
  - Ищем его оригинальные данные (title, price, url)
  - Отправляем в CreateAd API
  - Получаем новую ссылку
  - Генерируем персонализированный HTML
  - Отправляем ответ
↓
Профили остаются открыты
```

### 🔄 Поток данных

- `self.drivers_pool` - хранит открытые профили между этапами
- `self.sellers_data` - хранит все email и их оригинальные данные
- CreateAd API вызывается отдельно для каждого ответа
- Каждому ответу даётся уникальная ссылка

## v2.1 (2026-06-25)

### ✨ Новые возможности
- ✅ **Автосохранение API ключей** - все параметры автоматизации теперь сохраняются между запусками
- ✅ **Валидация email адресов** - используется regex для проверки корректности email
- ✅ **Retry логика для CreateAd API** - до 3 попыток с экспоненциальной задержкой
- ✅ **Лимит на количество писем с 1 почты** - настраивается в UI (по умолчанию 10)
- ✅ **Настройка задержки между первыми письмами** - в секундах

### 🐛 Исправления
- Исправлена обработка ошибок CreateAd API (rate limit 429)
- Добавлена валидация входных данных перед запуском
- Улучшена обработка невалидных email-ов от парсера
- Профили теперь открываются ПОСЛЕ парсинга (улучшенная логика)

### 📋 Добавлены примеры конфигов
- `config.json.example` - пример полной конфигурации
- `profiles.txt.example` - пример файла с профилями

### 🎨 UI/UX улучшения
- Новая вкладка "⚙️ Настройки" с информацией о версии
- Добавлены поля для лимита писем и задержки
- Улучшена типография интерфейса

## v2.0 (2026)

### Основные возможности
- Полная автоматизация рассылки через Gmail
- Интеграция с Dolphin Anty для использования профилей
- Парсинг email-ов с маркетплейсов
- Отправка первого письма всем пользователям
- Проверка ответов и отправка HTML писем с ссылками
- Персонализация писем через переменные {link} и {price}
- Spintax поддержка для вариативности текстов
- GUI на customtkinter с 6 вкладками
- Конфигурация сохраняется в JSON
- Система логирования в реал-тайм
